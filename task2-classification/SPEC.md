# 规格:植被/水体遥感分类

> 这份规格是"代码之前先写清楚要做什么、怎么验证"的契约。填完拿给导师过一遍。

## 目标

输入一景多波段遥感影像 `scene.tif`,训练一个像素级分类模型,把每个像素分到三类之一:
`0 = 其他`、`1 = 植被`、`2 = 水体`,并输出一张与原影像同尺寸的预测结果图 `prediction.tif`。

成功的定义不是"准确率高",而是**能证明模型真的学到了区分植被/水体的规律,而不是靠数据泄漏刷出来的虚高分数**。

## 输入

- `scene.tif`:4 个波段,波段顺序约定为
  - Band 1 = Red(红)
  - Band 2 = NIR(近红外)
  - Band 3 = Green(绿)
  - Band 4 = SWIR(短波红外)
- `labels.tif`:单波段,地面真值,取值 `{0, 1, 2}`。

各类占比:**运行时由 `train_eval.py` 自动统计并打印**(见验收标准),不在规格里写死。
原因:占比是判断"多数类基线"的依据,必须用真实数据现算,不能拍脑袋。

## 方法

- **模型:随机森林(RandomForest)。** 选它的理由:
  1. 像素级地物分类是表格型特征 + 强非线性边界,树模型天然适配;
  2. 对特征量纲不敏感,不需要做标准化;
  3. 自带特征重要性,能验证"模型是不是真的在用 NDVI/NDWI 这类有物理意义的特征";
  4. 训练快、可解释,适合这个阶段建立"能看懂"的习惯。
  作为对照可换 Logistic Regression 看线性可分程度,但主模型用 RF。

- **特征:** 4 个原始波段 + 3 个物理指数:
  - **NDVI** = (NIR − Red) / (NIR + Red) —— 植被对 NIR 强反射、对 Red 强吸收,NDVI 高即植被。
  - **NDWI** = (Green − NIR) / (Green + NIR) —— 水体对 NIR 强吸收,NDWI 高即水体。
  - **MNDWI** = (Green − SWIR) / (Green + SWIR) —— 用 SWIR 替代 NIR,对含建成区/阴影的影像辨水更稳。
  - 加指数的理由:它们是领域先验,直接把"植被/水体"的物理含义喂给模型,而不是让模型从原始波段里自己摸索。后续用特征重要性验证它们确实被用上了。
  - 分母加极小值 `eps` 防止除零。

## 训练/测试怎么切分(本任务核心)

⚠️ **像素之间不是独立的。** 遥感影像里地物成片分布,相邻像素几乎一模一样。

因此实现**两种切分,并把两种结果都报出来对比**:

1. **随机像素切分(random):** 把所有像素打乱再按比例切。
   —— 这是**错误但常见**的做法。测试集里几乎全是训练集像素的空间近邻,信息已经泄漏,准确率会虚高到接近完美。
2. **空间区块切分(spatial block):** 把影像切成网格区块,**整块**分给训练或测试,保证测试区块在空间上和训练区块不相邻。
   —— 这才是诚实的评估:模型必须泛化到"没见过的区域",分数通常会明显下降。

> 回答导师的问题"你怎么保证测试集没在偷看训练集":
> 用空间区块切分,测试块与训练块在空间上分离,相邻像素不会被拆到两边,因此测试集是模型真正没见过的区域。

## 评估指标(只看 accuracy 不够)

每种切分都输出:

- **多数类基线准确率**(全猜占比最大的那一类能得多少分)——模型必须明显超过它。
- **总体 accuracy**(只作参考,不单独下结论)。
- **balanced accuracy**(各类 recall 的平均,小类失败会立刻拉低它)。
- **混淆矩阵**(3×3)。
- **每一类的 precision / recall / F1**,重点看占比最小的**水体**。
- **macro-F1**(对类别不平衡更诚实,三类等权)。

额外两项诊断(应对"两种切分准确率可能一样高"的情况):

- **泄漏结构诊断**:测量每个测试像素到最近训练像素的距离分布。即使两种切分准确率相同,
  随机切分若有极高比例的"紧贴像素",结构上仍不可信。
- **弱特征对照实验**:故意把模型削弱到只用 1 个波段,演示"总体 accuracy 仍高、但小类 recall 崩"的假象——
  即"为什么 99% 准确率可能毫无用处"的活例子。

## 验收标准(可执行)

1. `python src/make_synthetic_data.py` 能生成 `data/scene.tif` 与 `data/labels.tif`(无真数据时用于跑通流程)。
2. `python src/train_eval.py` 能跑通,且打印:各类占比、多数类基线、两种切分各自的混淆矩阵与每类 precision/recall/F1、balanced accuracy、macro-F1。
3. **两种切分的 accuracy 对比 + 泄漏结构诊断都要报出来。** 若 random 与 spatial 的 accuracy 出现差距,需解释差距来自数据泄漏;**若两者准确率一样高(数据完全可分),则必须用泄漏结构诊断证明随机切分在结构上仍不可信(紧贴像素比例远高于空间切分)。**
4. spatial 切分下,模型的 macro-F1 **明显高于多数类基线**对应的 macro-F1(否则说明没学到东西)。
5. **弱特征对照实验能复现"总体 accuracy 高、但水体 recall 崩"的假象**,佐证"只看 accuracy 不够"。
6. 输出 `prediction.tif`,且预测图里水体成片、不是椒盐噪声。
7. 代码每一段我能逐段解释。
