# 卫星影像植被分析工具 - 规格文档（v2，已用 input.tif 实测元数据填充）

## 目标（一句话）
从多波段卫星影像中提取植被信息：计算 NDVI、生成植被掩膜、输出统计报告，结果保留地理参考，可直接在 QGIS / ArcGIS 中使用。面向遥感初学者与科研人员的命令行工具。

---

## 输入（以下均为 input.tif 实测值，非假设）

### 影像格式
- 格式: GeoTIFF，`driver = GTiff` ✓ 已确认
- 尺寸: **100 × 100 像素**（小图，可一次性读入；工具仍按分块写以支持大图）

### 波段信息（已从波段描述确认，无需猜测）
| 波段 | 描述(`src.descriptions`) | 含义 |
|------|--------------------------|------|
| 波段 1 | `Red`  | 红光 |
| 波段 2 | `NIR`  | 近红外 |

- 确认依据: `src.descriptions == ('Red', 'NIR')`。**红光=波段1，近红外=波段2。**
- 工具默认 `--red 1 --nir 2`，但允许命令行覆盖。

### 数据类型（★ 坑点 1）
- 数据类型: **uint16**（无符号 16 位整型）
- 值域: 波段1(Red) [0, 16009]，波段2(NIR) [0, 24626]
- **陷阱**: uint16 下 `NIR - Red` 若结果为负会下溢成大正数。
- **对策**: 计算前一律 `.astype('float64')`。

### NoData / 缩放系数（★ 坑点 2 —— 本任务最关键的坑）
- NoData 值: **0**（`src.nodata == 0.0`，全局标签也注明 `DN=0 is fill/nodata`）
- 缩放: `SCALE = 2.75e-05`, `OFFSET = -0.2`
- 全局标签原文: `reflectance = DN * SCALE + OFFSET; DN=0 is fill/nodata`
- 这是 **Landsat Collection 2 Level-2 地表反射率**的标准线性缩放。

**关键决策：必须先把 DN 转成反射率，再算 NDVI。**

原因（已用 analyze_edge_cases.py 实测验证）：
- NDVI = (NIR−Red)/(NIR+Red)。若 OFFSET=0，缩放系数会在分子分母约分掉，用 DN 直算也对。
- 但本影像 **OFFSET = −0.2 ≠ 0**，偏移量不会约分，用原始 DN 直算结果系统性偏低。
- 实测对比（像素 50,50）：NDVI(原始DN)=0.437 vs NDVI(反射率)=**0.787**，误差 0.35。
- 两种结果都落在 [−1,1] 内、图都画得出来 —— 这正是"结果错了图还正常"的典型陷阱。
- 正确公式: `reflectance = DN * 2.75e-05 + (-0.2)`，然后 NDVI 用反射率算。

实测反射率范围（有效像素）: Red [0.034, 0.240]，NIR [0.030, 0.477]，均为正、分母恒>0。

### 地理参考（实测）
- CRS: **EPSG:32650**（WGS84 / UTM zone 50N）
- 仿射变换: 像元 30m × 30m，左上角 (500000, 4000000)
- 边界: left=500000, bottom=3997000, right=503000, top=4000000

## 输出

1. **NDVI 栅格** `output_ndvi.tif`
   - 单波段，**float32**（保留 [−1,1] 精度）
   - nodata = **−9999.0**（区别于有效 NDVI；不能用 0，因为 0 是合法 NDVI 值）
   - 保留输入的 CRS 与 transform

2. **植被掩膜** `output_vegetation_mask.tif`
   - uint8：`1`=植被(NDVI>阈值)，`0`=非植被，`255`=无效/填充
   - 默认阈值 NDVI > 0.2，可由 `--threshold` 覆盖

3. **统计报告** `statistics.txt`
   - 有效/填充像素数、植被像素数与占比
   - 有效 NDVI 的 min/max/mean/std
   - 分级面积分布（按 30m 像元换算面积）

## 验收标准（可执行）

1. **手算对拍**: 取 3 个有效像素，按 `reflectance=DN*2.75e-05-0.2`、`NDVI=(NIR-Red)/(NIR+Red)` 纸笔/独立脚本计算，与工具输出差异 < 1e-5。
2. **独立重读**: 用 rasterio 重新读波段1/2 原始 DN，确认工具没读错波段。
3. **范围检查**: 所有有效 NDVI ∈ [−1, 1]。
4. **边界情况**:
   - DN=0 的填充像素 → 输出 nodata(−9999)，不能算出假植被。
   - 分母为 0 的像素 → 输出 nodata，不能产生 NaN/Inf。
5. **地理参考**: 输出与输入的 CRS、transform、尺寸完全一致（脚本断言 + QGIS 叠加目视）。
6. **统计合理性**: 植被占比、NDVI 均值落在物理合理区间。

## 非目标
1. 不做多时相/变化检测（单景）
2. 不做大气校正（输入已是 L2 地表反射率）
3. 不做云检测（无 QA 波段）
4. 不做 GUI（纯命令行）
5. 不做波段自动猜测（默认 Red=1/NIR=2，来自影像描述，可命令行覆盖）
6. 不支持非 GeoTIFF（HDF/NetCDF 请先转换）

---

## 关键坑点速查（交付时给导师看）
1. **uint16 下溢**: 算 NDVI 前必须转 float。
2. **OFFSET≠0 必须先转反射率**: 这是本影像最隐蔽的坑，用 DN 直算误差可达 0.35，且结果仍在 [−1,1] 看不出错。
3. **nodata 不能用 0**: 0 是合法 NDVI；输出 nodata 用 −9999。
4. **除零防护**: 分母为 0 输出 nodata，不留 NaN。
